# Full-Stack Web Developer Roadmap Skill

## Purpose

This skill provides a comprehensive learning path and curriculum for becoming a full-stack web developer, covering frontend, backend, databases, and DevOps.

## When to Use

Use this skill when:
- Starting web development journey
- Planning learning path
- Transitioning to full-stack role
- Identifying skill gaps
- Creating study curriculum
- Finding learning resources

## Learning Path

### Phase 1: Foundations (2-3 months)

| Topic | Technologies |
|-------|-------------|
| **Internet Basics** | How web works, HTTP/HTTPS, DNS |
| **HTML** | Semantic HTML, forms, accessibility |
| **CSS** | Selectors, flexbox, grid, responsive |
| **JavaScript** | Syntax, DOM, ES6+, async/await |
| **Git** | Version control, branching, merging |
| **Terminal** | Basic commands, navigation |

### Phase 2: Frontend (2-3 months)

| Topic | Technologies |
|-------|-------------|
| **Advanced CSS** | Animations, preprocessors (Sass) |
| **JavaScript Framework** | React, Vue, or Angular |
| **State Management** | Redux, Context API, Vuex |
| **Build Tools** | Webpack, Vite, npm/yarn |
| **Testing** | Jest, React Testing Library |
| **TypeScript** | Types, interfaces, generics |

### Phase 3: Backend (3-4 months)

| Topic | Technologies |
|-------|-------------|
| **Server-side Language** | Node.js, Python, or Go |
| **Web Framework** | Express, FastAPI, Django |
| **APIs** | REST, GraphQL |
| **Authentication** | JWT, OAuth, sessions |
| **Security** | OWASP Top 10, best practices |
| **File Handling** | Upload, storage (S3) |

### Phase 4: Databases (1-2 months)

| Type | Technologies |
|------|-------------|
| **Relational** | PostgreSQL, MySQL |
| **NoSQL** | MongoDB, Redis |
| **ORM** | Prisma, Sequelize, SQLAlchemy |
| **Query Optimization** | Indexes, query planning |

### Phase 5: DevOps & Deployment (1-2 months)

| Topic | Technologies |
|-------|-------------|
| **Containers** | Docker, Docker Compose |
| **CI/CD** | GitHub Actions, GitLab CI |
| **Cloud** | AWS, Vercel, Heroku |
| **Monitoring** | Logging, error tracking |
| **Performance** | Caching, CDN, optimization |

## For AI Assistants

When helping learners:

1. **Assess current level** - Beginner, intermediate, or advanced?
2. **Create learning plan** - Based on goals and timeline
3. **Recommend resources** - Courses, documentation, projects
4. **Suggest projects** - Hands-on practice
5. **Review progress** - Regular check-ins
